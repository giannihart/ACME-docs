## Overview

Lists organize information into readable bullet points or numbered sequences.

### Basic List Types

#### Unordered Lists

Create unordered lists using `-`, `*`, or `+` followed by a space:

```markdown
- First item
- Second item
- Third item
```

#### Ordered Lists

Create ordered lists using numbers followed by periods:

```markdown
1. First step
2. Second step
3. Third step
```

### Custom List Components

<Callout type="tip">
Most of the time, standard Markdown list syntax is simpler and more readable. Use custom components only when you need programmatic control or dynamic list generation.
</Callout>

### List Styling

Lists are automatically styled with:

- Consistent spacing between items
- Proper indentation for nested lists
- Matching colors with your theme
- Responsive margins and padding

### Best Practices

<Callout type="info">
**Use unordered lists when:**
- Order doesn't matter
- Listing features or options
- Creating navigation menus

**Use ordered lists when:**
- Describing sequential steps
- Showing rankings or priorities
- Order has semantic meaning
</Callout>

<Callout type="tip">
**Keep lists concise:**
- 3-7 items per list is ideal
- Use nested lists for subcategories
- Break long lists into sections with headings

**Format consistently:**
- Start each item with a capital letter
- End with periods only if items are complete sentences
- Keep parallel structure across items
</Callout>
