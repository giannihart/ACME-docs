## Overview

Steps display numbered, sequential instructions that guide users through procedures. Use them for tutorials, setup guides, and multi-step processes.

### Basic Usage

Wrap individual components inside a container. Step numbering is automatic.

<TabGroup>

Steps can contain any MDX components including code blocks, callouts, lists, and paragraphs.

### Component API

**Steps Container**

| Prop | Type | Required | Description | |------|------|----------|-------------| | `children` | `React.ReactNode` | Yes | Child `<Step>` components | | `className` | `string` | No | Custom CSS classes |

**Step Component**

| Prop | Type | Required | Description | |------|------|----------|-------------| | `title` | `string` | No | Step heading | | `children` | `React.ReactNode` | Yes | Step content | | `className` | `string` | No | Custom CSS classes |

<Callout type="info">
Step numbers and connecting lines are generated automatically. The final step omits the connecting line.
</Callout>

### Best Practices

<Callout type="tip">
- **Use descriptive titles**: Each step title should clearly indicate the action to perform
- **Keep steps focused**: Each step should represent a single, complete action
- **Add context**: Use callouts and explanatory text to provide additional context when needed
- **Include code examples**: Demonstrate commands and configurations with code blocks
- **Combine with other components**: Use CodeSnippet, Callout, and other MDX components to enhance clarity
</Callout>
